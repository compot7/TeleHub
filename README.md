# TeleHub - Электронные визитки и портфолио

Веб-приложение для создания и управления электронными визитками с портфолио работ и отзывами.

## Технологии

- **Backend**: Django 4.2+
- **Frontend**: Bootstrap 5, HTML5, CSS3, JavaScript
- **База данных**: SQLite (для разработки)
- **Python**: 3.10+

## Установка и запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Применение миграций

```bash
python manage.py makemigrations
python manage.py migrate
```

### 3. Создание суперпользователя

```bash
python manage.py createsuperuser
```

### 4. Запуск сервера разработки

```bash
python manage.py runserver
```

Приложение будет доступно по адресу: http://127.0.0.1:8000/

## Функциональность

### Публичная часть

- Просмотр визиток пользователей по URL: `/card/<username>/`
- Секции визитки:
  - **Обо мне**: биография, навыки
  - **Портфолио**: проекты с изображениями и описаниями
  - **Отзывы**: отзывы клиентов
  - **Контакты**: контактная информация, социальные ссылки, форма обратной связи, карта
- Скачивание визитки в форматах vCard и PDF
- Поиск визиток на главной странице

### Личный кабинет

- Редактирование профиля (аватар, фон, биография, контакты)
- Управление портфолио (добавление, редактирование, удаление проектов)
- Управление отзывами
- Управление социальными ссылками
- Просмотр сообщений из формы обратной связи
- Настройка темы (светлая/темная) и акцентного цвета

```
TeleHub/
├── telehub/          # Основные настройки проекта
├── cards/            # Приложение для визиток
│   ├── models.py     # Модели данных
│   ├── views.py      # Представления
│   ├── forms.py      # Формы
│   ├── urls.py       # URL-маршруты

├── templates/        # HTML-шаблоны
├── static/          # Статические файлы
├── media/           # Загруженные файлы
```

## Модели данных

- **CustomUser**: Расширенная модель пользователя с полями профиля
- **PortfolioItem**: Проекты портфолио
- **Testimonial**: Отзывы
- **SocialLink**: Социальные ссылки
- **ContactMessage**: Сообщения из формы обратной связи

## Административная панель

Доступна по адресу: `/admin/`

Используйте учетные данные суперпользователя для входа.

## Примечания

- Для работы карты Google Maps необходимо добавить API ключ в шаблон `card_detail.html`
- В production рекомендуется использовать PostgreSQL вместо SQLite
- Настройте `MEDIA_ROOT` и `STATIC_ROOT` для production-сервера
- Используйте HTTPS для защиты данных пользователей
